Code, Test, Refine, Repeat / Questions



What do you mean "Code, Test, Refine"?

  1. Write code
  2. Test to see how it looks when compared against the mock-up
  3. Refine/modify the code so that it looks closer and closer to the design you were given
  4. Repeat 1-3 as necessary until it looks like the desired page

Question: How do you know where to start testing?

A: Look for 'natural boxes' to determine structure

Question: How do you know what to refine so it starts to look more and more like the design?

A: Look for repeated styles and semantic elements. (Like h2 should be for the second biggest headings on the page, etc...) Think about how you can relate this to HTML elements, then
  1. Write your html
  2. Apply styles (from BIGGEST to smallest) (Site-wide elements to widgets)
  3. Fix things: Fix little things left over you can see right away, then check your site with multiple browsers, resize the window, etc... Repeat 1-3 until desired results achieved.

Q: What margin size gets the design closest to the mock-up?

A:Trick Question. There are multiple ways of doing this. The white space between boxes can be affected by the margins of both boxes, so removing whitespace can be achieved with margin manipulation. You change the right margin of the image to push text away, or leave the margin at 0 and, instead, apply a style to the margin of the text instead.

More On Dev Tools:

  • remember that developer tools can be a biog help during the code/test/refine cycle.
  • You can test things on the fly and quickly see how changes will affect your page without having to commit to anything. (e.g.: font size or family in h1, colour of background, etc)
  • You can also check code in the Sources Tab.
  • You can reset any changes just by clicking 'refresh'.

Comparing the Design

  • You can use Dev Tools to find the proper value of the margin for the image class.
  • Once you have refined it in Dev Tools, you can then save it to your CSS file.
  • If colours are not provided with the design mock-up/specs (usually provided in hex), you can ususally closely colour-match using the colour-picker tool to get the same information.
