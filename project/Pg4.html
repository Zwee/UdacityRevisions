Boxes and Containers



Container Elements:

Make web pages with compled structures possible. CSS is used to adjust different behaviours, to change how they display.

Span and Div

  • Both are regular elements & can have content
  • Their sole purpose it to contain things
  • Span is an INLINE container element
  • Div creates a 'BOX' container element

<span> and <div> are very similar, but <span> doesn't do any formatting itself. <div> can affect things because it defines logical breaks or divisions within the page(s). Meanwhile, the <span> tag just tells the browser to apply style rules to whatever's within the <span>.

Div:

<div> divides the content on screen using a line break above the <div> tag and another below the </div> tag.

The <div> tag defines logical divisions in your Web page. It's like a <p> tag, but it divides up the page(s) into larger sections than the <p> tag would. It also gives you the power to define big chunks of HTML.

<div> will help you label certain sections of your page(s) so you can make changes to them en masse with CSS. It's also helpful to remember that the <div> tag will break paragraphs, as it gets read as a <p> end/beginning, so you can have <p> within a <div>, you can't have a <div> within a <p>.

The main attributes of the <div> tag are: 1) style 2) class 3) id

Span:

<span> </span> items are displayed without line breaks. <div> defines logical divisions in your Web page. It's like a <p> tag, but it divides up the page(s) into larger sections than the <p> tag would. It also gives you the power to define big chunks of HTML.

The <span> has a lot in common with the <div> tag. It changes the style of the text that is contained within, however it won't change any of the content text until you first assign style attributes.

<span> is an in-line element and most often used to smaller chunks of in-line HTML. <div> is a block element (or 'block-line', which is basically the same as having a <br> before and after it) and used to group up larger bits of code.
