Use Github and start practising Version Control.

After making a lot of changes you might lose track of where you are. Version Control can save your butt if you think some code's gone missing, or if you can't recall what the last big change was.

Here's a link to Git's article about Version Control, what it is, how it can help you, and how to start implementing it.

Add style (like previously with the My App exercise mock-up) so you can see the boxes while you tweak your code.

It makes the various elements easier to see, and you just take this little bit of style out and the visual aid goes away. It also adds shading that we didn't use previously, but is a huge help to see overlap in boxes:

  • <style>
  • * {
  • margin: 8px;
  • border: red solid 2px;
  • background-color: rgba(0,0,0,0,15);
  • }
  • </style>
    • Note: The 'a' in 'rgba' is for Alpha (transparency), so the '15' value (above) is the transparency level.

Live Coding Session:

Hint for figuring out layout of content and structural choices before coding: map out your content on paper, or use Word/Google Docs/etc...
-the indenting and punctuation/headers/etc will start to give you ideas about where your content will look best. It's an organic, very visual approach and can make a river of content seem less daunting to place neatly on your page.
-It'll help you decide what's going to be nested, and why.

<p> bascially just tells the browser that you intend this to be a bunch of text (hence Paragraph) -It's OK to have 'special text' (like a href url...) to add links and images. It's still a paragraph. Trick: If some indentation is wonky, do this to make them all even: highlight the off-kilter lines, then hit TAB. They'll pop inline inwards. If you use shift + TAB they'll pop outwards. <div> basically just makes sure stuff inside has a box. It doesn't really have any special inate values.

Final Notes:

Question: Why is my picture not being styled correctly when I put it in a div and then style the div?

A: <img> is an element in itself, Add its style to it, not the div before it. Make sure you're putting the style where it needs to be and make sure you grok where it's supposed to go.

How to center an image in CSS:

  • .image-container {
  • text-align: center;
  • }
(Found using google search for 'how to center image in css. Remember: every element has to have its own box.)

Final reminder:

When going through your code, look at your CSS and ask "what do these things have in common, and how can I style just that?" and "How can I remove code that repeats itself?" When checking and troubleshooting, remember to start by checking for unclosed tags. It's a simple, common mistake. Don't Forget: If you're playing around with code and don't want to accidentally cut out something you might want to put back in later, just comment it out, then either delete it or uncomment it later once you've decided what you like.
