Adding CSS to Your Site / Understanding CSS Elements



3 Ways to Add CSS Styling:

  1. Write CSS in the <head> of your HTML
    • at top of HTML container, add "head"
    • Inside <head>, add a <style> element
    • Write your CSS in the <style> element, then put HTML for the rest of your page below
    • This is good for small projects, like from scratchpad.
  2. Have a a CSS file to accompany your HMTL file(s):
    • Write all structure HTML in 1 file (e.g.: main.html)
    • In separate file (main.css) put in all your CSS code
    • Add <head> at top of your HTML
    • Add <link> inside <head>
      • (Remember: <link> is a void tag)
    • Add these attributes to your <link>:
      • rel="stylesheet"
      • href="main.css"
  3. Write your style inline with your HTML
    • This is usually a bad idea as it leads to a LOT of repetition and makes for very unsightly code. However, you may still see code that uses this method, so it's good to be familiar with it. With this method, you modify the style attributes of every individual HTML element.
Some things to remember:

There are a lot of little mistakes you can and will make at some point that will cause any of these methods not to work. E.g.: writing code using 2)(above) with main.html and main.css in different folders on your computer.

Did you know: You will have a higher HTML:CSS code ratio than you might think.

This should make sense if you think of the difference this way: that HTML is the body, CSS is just the presentation. It only takes a little code to make big visual impacts.

Remember: When 'calling' (typing into code) an element (like body) does not get a . in front of it. Class names get a . in front when calling.

Understanding div, span, id, and class

<div> is a block.

Think: 'div-ides into boxes'. You can also think of "div"-ider tabs in a file cabinet or binder...they have labels, names, contents.

<span>'s

Are used for text decorations, and only change things inline. Spans do not affect their layout. **Span controls anything else inline that you want to change, without affecting the layout of the page.

Semantic Elements and Tags
  • Changing the visual properties of an element only changes how the element is displayed; it does not change the element itself or what kind of element it is.
  • Be sure to choose the correct type of element for cetain parts of a page, depending on what the purpose of that element is. (Using MyApp demo file as e.g.) The title at the top of the page is just the text giving the user an understanding of what's on the page, so you should be using a tag that provides such information.
  • Think of the elements to choose from when making a MS Word doc. (headers, footers, titles, headings, etc.) They all have specific attributes that are going to be consistent every time you use them. In HTML they're simply referred to as h1, h2, etc.
  • Search engines also use headings to index the structure and content of a web page.
