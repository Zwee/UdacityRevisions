Box Sizes: How to See Them, How to Change Them



Box Sizing Hint:

Consider this style:

{ box-sizing: border-box;}

This is something you would use to know how much space a given element will take, on any chosen screen. It calculates/changes the sizing of elements so you can see for yourself. This makes the calculation of the element size include both the border AND padding so that it is easier to make layouts. You set the width of the overall box, and the box will stay the same, regardless of padding or border tweaks you make later. Remember, when using this box-sizing trick, that the margin is NOT included in the size.

AND there's a catch: (Isn't there always a catch?)

This model is a fairly recent development, so some extra work is needed to be sure that older browsers can display your work. You will need to add what's called 'browser specific prefixes.'

  • They are:
    • -webkit (for Chrome and Safari)
    • -moz (for Firefox)
    • -ms (for IE)
  • (They all include the - in front of them.)

You just add them IN FRONT of the style definition. It looks like this:

  • * {
  • -webkit-box-sizing: border-box;
  • -moz-box-sizing: border-box;
  • -ms-box-sizing: border-box;
  • box-sizing: border-box;
  • }

Setting Size / Positioning

So far in the lesson, size has been shown in pixels, but there are a few ways to define size:
  • percentage: (responds/relies on screen size) The size WILL change depending on the size of the screen or how encompassing the boxes are.
  • pixels: (stays the same,regardless of screen size) The element WILL NOT change, no matter how big or small the display screen is. This can sometimes lease to a bad user experience.
  • max width: In this case, the element WILL resize and become smaller when necessary, but it will NOT go larger than the set max width.

About Positioning:

  • Good News: There is a lot of power in CSS to position elements
  • Bad News: This is probably one of the most complicated concepts about CSS

One powerful technique to use is called Flex Box Layout

'Flexbox' means 'flexible box' and it provides an efficient way to lay out, align, and distribute space among items in a container ('div').

Using our MY APP exercise: In our mock-up, we have our image next to our description. If you'd like several elements to be next to each other, you can change the 'display' attribute of the Parent Container to value 'flex'.

In our My App exercise, we have our parent div class 'app'. In <div class="app"> (clicked to highlight in Dev. Tools.) we can click to R side of dev tools screen, under 'styles' tab, next to 'element.style {...' we'll add the rule 'display: flex' to the element style, and we should get the child elements trying to align next to each other. For it to work, you also have to give the items a size that is SMALLER than 100% (default).

E.g.:
  • element.style {
  • display: flex;
  • }
