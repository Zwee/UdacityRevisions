Sizing Elements, Flexbox, Adding Images



Learning how to be Flexible (or display purposes only.)

On a previous page, we saw the rule 'display: flex' added to 'element.style', which adds the CSS directly to the 'div' element on the left-hand side of the development tools. In this, and future mentions, this CSS rule is in the main .css file, 'style.css:'

  • .app {
  • display: flex;
  • }
The rule works as-is with Chrome 29+, IE 11+, and Mozilla 28+. To support Safari, the '-webkit-' prefix MUST be added: (Note the class selector, as mentioned earlier.)
  • .app {
  • display: -webkit-flex;
  • display: flex;
  • }

Box Sizing

4 main points:
  1. HTML elements are boxes and each box has 4 components.
  2. Because there are so many components to a box, it can often be hard to get the size of a box just right.
  3. Two techniques you can use to deal with sizing issues:
    • Set sizes in terms of percentages vs. pixels
    • Set box-sizing attribute to border-box for every element (mentioned above)
  4. Different browsers differ slightly, one to the other, so this can cause some to display the same code differently.

Box Positioning

  • <div>'s are BLOCK elements (as opposed to inline) so, by default, they take up the entire width of a page.
  • Adding the rule 'display: flex;' to the appropriate CSS will over-ride this behaviour and let <div>'s appear next to each other.

Adding Images

  • You have to store the image somewhere that the page can access it.
  • You can add the image to the page with the 'img' tag and providing the source ('src') of the image.
  • You should always make sure to add your 'alt' attribute, to display alternate info in case your image doesn't display.
